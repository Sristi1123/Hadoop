--> Step 1 — Grant full permissions to everyone (chmod 777)
hdfs dfs -chmod 777 /BACKUP/file1.txt

Step 2 — Verify the permission change
hdfs dfs -ls /BACKUP

--> Step 3 — Set a custom permission (chmod 527)
hdfs dfs -chmod 527 /BACKUP/file1.txt

** Apply chmod to an entire directory (recursive)
# Change permissions on a directory and everything inside it
hdfs dfs -chmod -R 755 /BACKUP

            The -R flag applies the permission change recursively to the directory
            itself and all files and subdirectories inside it.
            Without -R, only the directory itself is changed — not its contents.

✅ Octal Permission (Easy Understanding)

r w x = 7 → read + write + execute (full access)
r w - = 6 → read + write
r - x = 5 → read + execute
r - - = 4 → read only
- w x = 3 → write + execute
- w - = 2 → write only
- - x = 1 → execute only
- - - = 0 → no permission

--> 📌 chown (change owner)
👉 Used to change owner and group of a file/directory 

--> Change owner only: hdfs dfs -chown me /BACKUP/file1.txt
--> Change owner + group: hdfs dfs -chown sn:supergroup /BACKUP/file1.txt
--> hdfs dfs -ls /BACKUP  (check from this command)
--> Change group only: hdfs dfs -chown :biggroup /BACKUP/file1.txt
--> Apply to all files (recursive): hdfs dfs -chown -R srist:supergroup /BACKUP

📌 chgrp (change group)
👉 Used to change only the group of a file or directory

--> hdfs dfs -chgrp group /BACKUP/file1.txt  --> Changes group of file1.txt to group

🔹 Recursive (all files inside folder): hdfs dfs -chgrp -R biggroup /BACKUP

👉 Trick to remember:
read (r) = 4
write (w) = 2
execute (x) = 1


✅ Symbolic Form
First 3 → Owner
Next 3 → Group
Last 3 → Others
Example: rwx   r-x   r--

👉 Means:
Owner → read, write, execute
Group → read, execute
Others → read only

--> hdfs dfs -chmod u+rwx,g+rx,o+r /BACKUP/file.txt

📌 ls -l (10 bits format)
Total 10 characters
1st bit → type
- → file
d → directory

Next 9 bits → permissions
1–3 → user (owner)
4–6 → group
7–9 → others

📌 Default permissions
❌ Not always 777
✔️ Depends on umask
Files → 666 → after umask → usually 644
Directories → 777 → after umask → usually 755

📌 Meaning of “mask” and “unmask”
Mask → hiding/removing permissions
Unmask → permissions that are left after masking

👉 Example:
Default = 777
umask = 022
Final = 755

📌 Example
666 (rw-rw-rw-) ❌ → cannot enter directory
111 (--x--x--x) ✔️ → can enter but not see files
755 (rwxr-xr-x) ✅ → best (enter + read)

📌 Final understanding
✔️ Yes, execute (1) is required for directory access
❗ But only 111 is not enough in real use

👉 Short (exam)
Directory must have execute (x) permission
Without x → cannot access folder

chmod dir → affects only directory
Controls access to folder
Use -R → for all files inside

📌 Basic Difference
🔹 chown
👉 Changes owner (user) + group
hdfs dfs -chown user:group file

🔹 chgrp
👉 Changes only group
hdfs dfs -chgrp group file
